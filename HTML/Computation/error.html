Error

There are several types of error in the approximations used in numerical analysis:

  • Computational Error - difference between exact and approximate functions for same input
    • Truncation Error - difference between a true result and the result given by a function
    • Rounding Error - difference between result given from a function using true arithmetic, and result given using rounded inputs
  • Data Error - difference between exact function values due to error in input value(s)
  • Absolute Error - difference between the approx value and the true value
  • Relative Error - absolute error divided by the true value (can be expressed as percentage when multiplied by 100)


Additionally, problems can be classified as insensitive (well-conditioned) or sensitive (ill-conditioned): that is to say, for a sensitive problem, a small relative change in the input data would lead to a large relative change in the output of the function. This is defined by a condition number. We can also use backward error analysis instead of forward error analysis to analyze error propagation. With this method we:
1. Consider the approx sol to be the exact sol
2. Determine how large a modification to OG problem is required to get that sol

  • Sensitivity - the degree to which the input data affects the difference in output data
  • Conditioning - the quantative/numerical measure of sensitvity

Matrix Error

The error bounds of a matrix can be derived from the condition number. Formally, the relative change in the solution to a linear system Ax=b is bounded by the condition number, multiplied by the relative change in the input data. This measurement quantatively defines the sensitivity of the system.
