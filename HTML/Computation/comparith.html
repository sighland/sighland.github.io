Computer Arithmetic and Floating-Point Numbers

  • Β - base/radix
  • t - precision
  • e - exponent
  • [L,U] - exponent range

A floating-point system is normalized if the leading digit d0 is always nonzero (unless to num represented is 0). Ergo, the mantissa m is always 1 ≤ m < Β.
  • Normalization - a floating-point system (FPS) is normalized if the leading digit d0 is always nonzero
  • Underflow Level (UFL) - smallest positive normalized floating-point number. The first digit is 1 followed by 0s.
  • Overflow Level (OFL) - largest positive normalized floating-point number. Each digit of the mantissa is Β-1
  • Rounding - Process of choosing a nearby floating-point number fl(x) to approximate a real number x. This leads to rounding rules.
    • Chop (round-toward-0) - truncated to the t-1 digit
    • Round-to-nearest (round to even) - fl(x) is nearest floating-point to x; if tie, take even
  • Machine Precision (unit roundoff) - determines the accuracy of a floating-point system, as well as the max possible relative error

In a floating point system, there are no floating-point numbers between 0 and ΒL. If we allow leading digits to be zero (only when at UFL), then there can be floating-point numbers here. These extra added numbers are called subnormal or denormalized. These have a lower precision t than normalized numbers. A system like this is said to have gradual underflow.

Floating-Point Arithmetic

In addition, the exponents of a FPN must be shifted. If the sum contains more than t digits, a shift occurs and we lose digits.
Conversely, the product of two FPN need not be shifted, but may have up to 2t digits; hence, we can also lose a lot, since we can have t digits max. It should also be noted that addition/multiplication of FPNs are commutative but not associative. Because floating-point arithmetic need not obey the laws of real arithmetic, forward error analysis can be difficult -- hence, backwards error analysis works better, as it can use real arithmetic.

In subtraction, the result will have fewer than t digits, so it is always fully representational. However, this also leads to cancellation if the operands are too close.
